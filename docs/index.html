1. JavaFX Core & Layouts

Concepts & Classes Detailed Explanation & Code

How it works: Every JavaFX app inherits from Application. The launch() method starts the app, which eventually calls start(Stage). The Stage is your actual window, and the Scene is the canvas inside it. You load your visual design (built in Scene Builder) using FXMLLoader.

Layouts: Elements are organized using containers. HBox lines widgets up horizontally, while VBox stacks them vertically. SplitPane gives you a draggable divider between two areas, and ScrollPane adds scrollbars if your content is larger than the screen. 
// 1. Layout Methods (Spacing & Sizing)
HBox hbox = new HBox();
hbox.setPadding(new Insets(15, 12, 15, 12)); // Inner padding inside the box
hbox.setSpacing(10); // Space between widgets inside the box
hbox.getChildren().addAll(node1, node2); // Adding elements

// 2. Navigation Layout
SplitPane split = new SplitPane();
split.setOrientation(Orientation.VERTICAL); // Adds a draggable vertical divider

// 3. Application Lifecycle Loading
Parent root = new FXMLLoader(getClass().getResource("login.fxml")).load();
primaryStage.setScene(new Scene(root, 500, 300));
primaryStage.setTitle("My App"); 
primaryStage.show();

Interacting with the user: TextField accepts text, while PasswordField replaces typed characters with dots. ComboBox is a dropdown list (which can be made searchable/editable). 3. Java Collections Core API

Concepts & Classes Detailed Explanation & Code
Modification:
add() addAll()
remove() removeAll()
retainAll() clear()

Inspection:
contains() containsAll()
isEmpty() size()
toArray() iterator()
The Foundation: The Collection interface guarantees that ALL lists, sets, and queues share these core methods.
• contains() searches the list to see if a specific item exists.
• retainAll() is powerful: you give it a second collection, and it deletes everything in your first collection EXCEPT the items that exist in both (intersection).
• removeAll() deletes a whole batch of items at once.
• isEmpty() checks if the list is blank, and size() counts the items. 
Collection<String> items = new ArrayList<>();
items.add("Apple"); 
items.addAll(otherCollection); // Merges another collection into this one

boolean hasApple = items.contains("Apple"); // Returns true if "Apple" is found
boolean hasAll = items.containsAll(otherCollection); // Checks if all exist

items.retainAll(otherCollection); // Keeps ONLY items that also exist in otherCollection
items.removeAll(otherCollection); // Removes items that exist in otherCollection

items.clear(); // Empties the entire collection instantly
boolean empty = items.isEmpty(); // Returns true if size is 0
int count = items.size(); // Returns total item count

4. Data Structures (Lists, Sets, Maps)

Concepts & Classes Detailed Explanation & Code

Lists vs Sets:
• Lists keep items in the exact order you add them. You can grab an item by its index number (get(0)).
• Sets are strict: they automatically reject duplicate items. A TreeSet is magical because it automatically sorts your data (alphabetically or numerically) as you add it! 
// LISTS: Access by index, duplicates allowed
List<String> list = new ArrayList<>(); 
list.add(0, "First"); // Inserts exactly at position 0
list.set(0, "Replaced"); // Replaces element at position 0
int pos = list.indexOf("Replaced"); // Finds the index number of an item

// SETS: Rejects duplicates, TreeSet automatically sorts!
Set<String> uniqueNames = new TreeSet<>();
uniqueNames.add("Zebra");
boolean added = uniqueNames.add("Apple"); // Returns true
boolean addedAgain = uniqueNames.add("Apple"); // Returns false! Set size is 2.
Maps:
HashMap TreeMap
put() get() remove()
containsKey() values()
entrySet()
Dictionaries (Maps): A Map doesn't use index numbers. Instead, it links a "Value" to a unique "Key" (like linking a Zip Code to a City Name). You use put() to store the pair, and get(key) to instantly retrieve the value later without having to search through the whole list. 
Map<String, Integer> map = new HashMap<>();
map.put("Paris", 75); // Adds or replaces key-value
map.putAll(otherMap); // Merges another map into this one

int zip = map.get("Paris"); // Instantly retrieves 75
boolean hasKey = map.containsKey("Paris"); // Checks if key exists

// Iterating safely over a Map
for (Map.Entry<String, Integer> entry : map.entrySet()) {
    System.out.println(entry.getKey() + " -> " + entry.getValue());
}

5. Iteration, Generics & Sorting

Concepts & Classes Detailed Explanation & Code
Generics:
<T> <K, V>
<? extends T> (Upper Bound)
<? super E> (Lower Bound)
Type Safety: Generics (<T>) prevent crashes by forcing a collection to hold only one type of data. Wildcards (? extends Employe) are flexible: they let a method accept a list of Employe OR any of its sub-classes. However, to prevent data corruption, Java locks these wildcard lists to "Read-Only"—you cannot add to them! 
// Generic Class allowing any Object type 'T'
public class Box<T> { private T item; }

// Upper Bound (Read-Only): Accepts 'Employe' or its subclasses (like Vendeur)
public void calculate(ArrayList<? extends Employe> list) { 
    // You can READ, but you CANNOT use list.add() here!
}
Iteration:
Iterator ListIterator
forEach(lambda) removeIf()
hasNext() remove()
Safe Deletion: If you try to use list.remove() while looping inside a standard for-each loop, your app will crash. To safely delete items while looping, you must explicitly use an Iterator, or use the cleaner modern Java Lambda function: removeIf(). 
// SAFE METHOD 1: Manual Iterator (Safe deletion during iteration)
Iterator<String> it = list.iterator();
while(it.hasNext()) {
    if(it.next().equals("DeleteMe")) { 
        it.remove(); // This is safe! Never use standard list.remove() here.
    } 
}

// SAFE METHOD 2: Functional Lambda (Does the exact same thing in one line)
list.removeIf(elem -> elem.equals("DeleteMe"));
Sorting Objects:
Comparable<T>
compareTo()
Comparator<T>
Collections.sort()
Custom Sorting Rules: Java doesn't know how to sort a custom Eleve (Student) object.
• Comparable: You write this inside the Eleve class to define the natural default sort (e.g., sort by average grade).
• Comparator: You write this outside the class as a separate tool when you need a secondary sorting rule (e.g., sort by Age instead of Grade). 
// 1. COMPARABLE (Internal Natural Sort by Grade)
public class Eleve implements Comparable<Eleve> {
    public double moyenne;
    public int compareTo(Eleve autre) {
	return Double.compare(this.moyenne, autre.moyenne);
	}
}

// 2. COMPARATOR (External Custom Sort by Age)
class TriParAge implements Comparator<Person> {
    public int compare(Person p1, Person p2) { 
	return Integer.compare(p1.getAge(), p2.getAge()); 
	}
}

// Execution
Collections.sort(listeEleves); // Sorts naturally via Comparable (Grade)
Collections.sort(listePersonnes, new TriParAge()); // Sorts via Comparator (Age)
